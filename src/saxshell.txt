------------------------------------------------------------------------------------------
Project Purpose
---------------
SAXShell is a software framework for extracting chemically meaningful
solute-centered cluster ensembles from molecular dynamics trajectories
and using those extracted cluster families as interpretable building
blocks for small-angle X-ray scattering (SAXS) model construction,
initial deterministic fitting, and Bayesian refinement.

Core Idea
---------
Rather than modeling SAXS directly from an entire molecular dynamics box
as one opaque object, SAXShell reduces the trajectory into a set of
interpretable local structural motifs.

These motifs are:
- solute-centered atom networks
- coordinated solvent shells
- grouped cluster families sorted by stoichiometry
- optionally later classified by topology, coordination motif, or other
  descriptors

These cluster families then become the structural basis for building
multicomponent SAXS models.

High-Level Workflow
-------------------
1. Load a molecular dynamics trajectory
2. Select a frame range and stride
3. Extract individual frames
4. For each frame:
   a. export or access the frame as XYZ
   b. identify molecules using a user-defined molecule reference file
   c. assign atoms into distinct molecules
   d. convert the frame into a structured PDB-like representation
5. For each frame PDB:
   a. identify solute-centered atom networks
   b. identify coordinating solvent molecules
   c. extract solute-plus-solvation-shell clusters
6. Collect clusters across all frames
7. Sort clusters into cluster families by stoichiometry
8. Build structural distributions and summary statistics
9. Compute SAXS components for cluster families
10. Build an initial multicomponent SAXS model
11. Perform an initial deterministic fit using lmfit
12. Use the lmfit result to initialize a secondary Bayesian fit using
    pyDREAM
13. Save and analyze final fit results

Conceptual Layering
-------------------
SAXShell should be separated into four major layers:

1. Structural Extraction Layer
   - trajectory loading
   - frame handling
   - molecule identification
   - cluster extraction

2. Cluster Dataset Layer
   - cluster family grouping
   - stoichiometric sorting
   - representative cluster selection
   - structural statistics and distributions

3. SAXS Modeling Layer
   - scattering component generation
   - model data preparation
   - multicomponent SAXS model construction

4. Fitting and Inference Layer
   - deterministic model prefit with lmfit
   - Bayesian sampling and refinement with pyDREAM
   - diagnostics and posterior analysis

Detailed Workflow Description
-----------------------------

Stage 1. Trajectory Loading
---------------------------
Input:
- MD trajectory file
- topology or metadata if needed
- user frame selection settings

Purpose:
- read the trajectory into an internal representation
- identify available frame indices
- expose frame subsetting tools

Output:
- trajectory object
- frame index list
- metadata for downstream extraction

Stage 2. Frame Selection and Extraction
---------------------------------------
Input:
- trajectory object
- frame range
- stride

Purpose:
- select a subset of frames for analysis
- extract individual frame snapshots

Output:
- Frame objects or temporary XYZ snapshots

Stage 3. Molecule Identification
--------------------------------
Input:
- XYZ frame
- molecule reference file

Purpose:
- define molecules distinctly within each frame
- assign atoms to molecular identities
- distinguish solvent molecules, solute molecules, ions, ligands, etc.

Output:
- MoleculeAssignment for each frame

Stage 4. Frame Conversion
-------------------------
Input:
- frame data
- molecule assignments

Purpose:
- convert each frame into a PDB-like structured form
- preserve residue, molecule, and atom identity needed for extraction

Output:
- frame PDB representation

Stage 5. Cluster Extraction
---------------------------
Input:
- frame PDB
- user extraction settings

Purpose:
- identify solute-centered atom networks
- identify coordinating solvent molecules around those networks
- extract solute-plus-solvation-shell clusters

Output:
- Cluster objects for each frame

Stage 6. Cluster Family Grouping
--------------------------------
Input:
- all Cluster objects across all frames

Purpose:
- group extracted clusters into family types
- primary grouping by stoichiometry
- later extensible to topology, coordination motif, symmetry, etc.

Output:
- ClusterFamily objects
- ClusterEnsemble object

Stage 7. Structural Analysis
----------------------------
Input:
- ClusterEnsemble

Purpose:
- analyze structural distributions across cluster families and members

Examples:
- stoichiometry histograms
- bond length distributions
- bond angle distributions
- coordination number distributions
- species-resolved statistics
- representative cluster structures

Output:
- plots
- summary tables
- structural metrics

Stage 8. SAXS Component Generation
----------------------------------
Input:
- ClusterEnsemble
- scattering settings
- q-range

Purpose:
- compute SAXS contributions for cluster families
- create a component library that maps cluster family identity to
  scattering curve

Output:
- ScatteringComponent objects
- ComponentLibrary

Stage 9. SAXS Model Construction
--------------------------------
Input:
- ComponentLibrary
- experimental SAXS data
- model constraints and settings

Purpose:
- define a multicomponent SAXS model
- assign component weights, scales, bounds, and backgrounds
- prepare a reusable SAXS model data object

Output:
- SAXSModelData
- SAXSModel

Stage 10. Initial Deterministic Fit with lmfit
----------------------------------------------
Input:
- SAXSModelData
- SAXSModel

Purpose:
- generate a sensible initial fit
- estimate model weights and nuisance parameters
- identify parameter degeneracy and obvious model issues
- create an initialized model state for Bayesian refinement

Output:
- LMFitResult
- initialized SAXSModel

Stage 11. Secondary Bayesian Fit with pyDREAM
---------------------------------------------
Input:
- initialized model
- LMFitResult
- priors and DREAM sampling settings

Purpose:
- perform posterior exploration
- quantify uncertainty
- identify parameter correlations
- generate posterior predictive model behavior

Output:
- DREAMFitResult

Recommended Responsibilities by App
-----------------------------------

extract_clusters
- owns trajectory parsing and structural extraction
- should not perform fitting

analyze_clusters
- owns structural characterization of extracted ensembles
- should not construct mathematical models

build_components
- owns conversion of cluster families into scattering components
- should not perform fitting

build_model
- owns model configuration and assembly
- should not perform long inference

prefit_model
- owns deterministic initialization
- should not perform posterior sampling

dream_fit_model
- owns Bayesian refinement and uncertainty quantification

Recommended Design Principles
-----------------------------
1. Keep structural extraction separate from SAXS model construction
2. Keep SAXS model construction separate from fitting
3. Keep deterministic initialization separate from Bayesian inference
4. Use explicit data objects between stages
5. Avoid a single monolithic "god object"
6. Make each stage reloadable from disk
7. Make cluster family identity central to the software design
8. Treat ClusterEnsemble and SAXSModelData as stable bridge objects

What Not To Do
--------------
Do not build one giant object that stores:
- raw frames
- molecule assignments
- extracted clusters
- cluster distributions
- scattering components
- model parameters
- fit results
- posterior samples

That would become difficult to maintain, hard to serialize cleanly, and
hard to reason about.

Recommended Stable Bridge Objects
---------------------------------
1. ClusterEnsemble
   - stable output of structural extraction

2. ComponentLibrary
   - stable output of scattering generation

3. SAXSModelData
   - stable output of model preparation

4. LMFitResult
   - stable output of deterministic initialization

5. DREAMFitResult
   - stable output of Bayesian refinement

One-Sentence Summary
--------------------
SAXShell should convert molecular dynamics trajectories into chemically
meaningful solute-centered cluster ensembles, organize those ensembles
into interpretable cluster families, use those families to construct a
multicomponent SAXS model, initialize the model with an lmfit prefit,
and then refine parameter uncertainty and posterior behavior with
pyDREAM.
